如果需要修改配置，使用workflow命令：


可以修改：
1. 人设
2. 默认图片风格
3. 账号
4. 默认图片数量
5. 自动优化开关
6. 自动发布开关

## 重置配置

如果需要重新开始：

```bash
/xhs:workflow --reset-config
```

这会清除所有配置，下次使用时重新设置。

## 常见问题

### Q: 配置保存在哪里？

A: 配置保存在插件根目录的 `.user_config.json` 文件中。

### Q: 可以有多个账号吗？

A: 目前不支持多账号，建议为每个账号创建不同的配置文件（未来版本会支持）。

### Q: 如何查看当前配置？

A: 运行 `/xhs:workflow` 会显示当前配置。

### Q: 配置会丢失吗？

A: 配置文件是持久化的，除非你手动删除或运行 `--reset-config`。

### Q: 可以手动编辑配置文件吗？

A: 可以，直接编辑 `.user_config.json` 文件即可。

### Q: 忘记了配置了什么怎么办？

A: 运行 `/xhs:workflow` 会显示当前配置摘要。

## 提示和技巧

### 1. 合理选择图片数量

- **测试阶段**: 生成1张查看效果
- **正式发布**: 生成3-7张

### 2. 调整自动优化

如果不喜欢自动优化，可以关闭：

```bash
python3 $CLAUDE_PLUGIN_ROOT/scripts/config_manager.py set auto_optimize false
```

### 3. 批量创作（未来功能）

未来版本会支持：

```bash
/xhs:workflow --batch 3
# 一次性创作3篇帖子
```

### 4. 定时发布（未来功能）

未来版本会支持：


## 对比：旧流程 vs 新流程

```bash
# 1. 分析热榜
/xhs:analyze-trending
# - 选择平台
# - 选择人设
# - 等待API

# 2. 创建帖子
/xhs:create-post

# - 选择选题
# - 确认账号
# - 确认创建

# 3. 优化帖子
/xhs:optimize-post --file "路径"
# - 选择模式
# - 选择优化项
# - 选择标题

# 4. 生成配图
/xhs:generate-images --post "路径"
```
